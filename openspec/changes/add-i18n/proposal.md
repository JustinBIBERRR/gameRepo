# Change: 项目国际化（i18n）

## Why

当前所有界面文案均为硬编码中文，无法服务非中文用户。引入 i18n 后，平台可支持多语言（如中文、英文），便于公司海外团队或国际化团建场景使用，并保持后续扩展其他语言的能力。

## What Changes

- 引入 Vue 生态常用的 i18n 方案（如 vue-i18n），在应用层提供当前语言与翻译消息的绑定。
- 将界面中所有用户可见的文案抽离为按语言组织的消息键值（locale 文件），至少支持中文（默认）与英文。
- 提供语言切换入口（仅在导航栏），用户切换后界面即时切换语言，且偏好持久化到本地（如 localStorage）。
- 游戏配置中的标题、描述等展示文案改为从 i18n 消息中读取，与组件内文案统一管理。
- 不改变现有路由、业务逻辑与数据存储结构；仅将“展示用字符串”改为通过 i18n 解析后的结果。

## Impact

- Affected specs: 新增 `i18n`（国际化能力）；可选在 `platform-core` 中补充“首页/导航文案来自 i18n”的说明（本提案以新增 i18n 能力为主，不强制修改 platform-core 现有条文）。
- Affected code:
  - 应用入口：`src/main.ts` — 注册 i18n 插件并设置初始语言。
  - 新增：`src/locales/` 或 `src/i18n/` — 语言包（如 `zh-CN.json`、`en.json`）及 i18n 初始化（如 `index.ts`）。
  - 全局布局/导航：`src/App.vue`、`src/components/Navigation.vue` — 增加语言切换 UI（或入口）。
  - 配置与视图：`src/config/games.ts`、`src/views/Home.vue`、`src/views/Settings.vue` 及各游戏视图、通用组件 — 将硬编码中文替换为 i18n 键调用（如 `$t('key')` 或 `useI18n().t`）。
  - 依赖：新增 `vue-i18n`（或所选 i18n 库）。

## 澄清结论（需求确认）

以下为与产品确认后的边界与策略，实现与验收以此为准：

| 维度 | 结论 |
|------|------|
| **游戏数据** | 不纳入平台 i18n。游戏内展示的数据（城市名、英雄名、电影名、看图猜物物品名等）由管理者在设置中配置，平台只做界面国际化。 |
| **首次访问语言** | 不检测浏览器语言，一律默认中文（zh-CN）；用户手动切换后再持久化。 |
| **语言切换入口** | 仅在导航栏（Navigation）提供一处语言切换，全局只此入口。 |
| **缺失 key 表现** | 先回退到 zh-CN 同一 key，若仍无则显示 key 字符串；与 vue-i18n 的 fallbackLocale 一致。 |
| **复数形式** | 首轮用插值（如 `{ count }`）显示数字，不实现 "1 win" / "2 wins" 等复数规则；后续可再加。 |
