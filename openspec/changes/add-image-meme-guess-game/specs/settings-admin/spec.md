## ADDED Requirements

### Requirement: 看图猜梗数据 Schema
系统 SHALL 为看图猜梗游戏定义数据 Schema，每题包含图片与答案，不包含提示开关或提示内容字段。

#### Scenario: 看图猜梗数据 Schema 定义
- **WHEN** 系统定义看图猜梗数据 Schema
- **THEN** 包含字段：id（主键）、image（图片，必填，**支持在线 URL 与本地上传文件**两种方式）、answer（答案，必填）、answerVariants 或 aliases（可选，用于别名匹配）
- **AND** 不包含「提示」或「开启提示」类字段
- **AND** 管理员在设置的数据管理中进行增删改查时，仅编辑图片与答案相关字段
- **AND** 图片格式与单张大小限制由实现决定合理上限

#### Scenario: 看图猜梗配置无提示开关
- **WHEN** 管理员在设置页面进入看图猜梗配置
- **THEN** 游戏参数区域不提供「显示初始提示」或类似提示开关
- **AND** 数据管理表单中不包含提示内容、提示列表等字段
- **AND** 题目仅由「图片 + 答案」构成

#### Scenario: 看图猜梗可选最大尝试次数
- **WHEN** 管理员在看图猜梗游戏配置页设置游戏参数
- **THEN** 默认不限制每题尝试次数（可一直猜直到猜对）
- **AND** 可选配置「最大尝试次数」（如 3–10 次）；配置后每题用尽次数则判负，并进入下一题或再来一局
- **AND** 未配置或配置为「不限制」时，行为与默认一致

#### Scenario: 看图猜梗可选倒计时
- **WHEN** 管理员在看图猜梗游戏配置页设置游戏参数
- **THEN** 支持可选倒计时（与电影等游戏一致）：可启用/禁用，可配置每题或每局时长（如 1–60 分钟）
- **AND** 设置即时保存，下一局或重新进入游戏后生效

#### Scenario: 看图猜梗题目条数上限
- **WHEN** 管理员在看图猜梗数据管理中查看或新增题目
- **THEN** 题目条数上限为 **500 条**，与 settings-admin 其他游戏一致
- **AND** 达到上限时「新增」按钮禁用并提示「已达 500 条上限」

#### Scenario: 看图猜梗默认题目数据
- **WHEN** 用户首次使用看图猜梗或使用默认数据
- **THEN** 系统提供 **2 条示例题目**，开箱可玩：
  - 题目一：图片 `img/guessPic/pigGuess.png`，谜底 **「二锅头」**
  - 题目二：图片 `img/guessPic/roubaoGuess.png`，谜底 **「书包」**
- **AND** 管理员可在此基础上切换为自定义数据并增删改查

#### Scenario: 看图猜梗数据过期
- **WHEN** 应用启动或管理员访问看图猜梗配置页
- **THEN** 看图猜梗自定义数据参与**数据过期检查**（与 settings-admin 其他游戏一致）
- **AND** 过期后自动清除自定义数据并恢复默认数据，并提示管理员
